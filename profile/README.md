## Why This Log seq Exists

Most note-taking apps force you to choose between visual clutter (Notion) or markdown limitations (Obsidian). For **outline thinking**, they fail because they treat lists as text, not data. **Log seq** solves this fragmentation by making every dash and indent a first-class citizen in a database-like outline engine.

**Log seq** solves **one specific problem**: the friction of capturing hierarchical thoughts quickly. Whether you are journaling daily, managing a project, or researching, **Log seq** keeps your information structured as nested lists. No folders, no tags required—just pure **Log seq** outline power.

If you are tired of wrestling with markdown syntax just to create a sub-point, or if you miss the outliner experience of Workflowy but want an **open-source Obsidian alternative**, **Log seq** is for you.

## Quick Start with Log seq

1. **Get Log seq** — Download the portable `.exe` or AppImage for **Log seq** (no installation required for the Windows version).
2. **Launch** — Double-click `Log_seq.exe`. The **Log seq** main window opens in under 1.5 seconds.
3. **Create your first journal entry** — Type `-` and a space. Write "Morning thoughts" in **Log seq**.
4. **Create a nested outline** — Press `Tab` to indent. Type `- Wake up at 7am`. Press `Enter`. Type `- Make coffee`. See how **Log seq** keeps the hierarchy.
5. **Use daily journal** — Press `Ctrl + Shift + D` in **Log seq**. **Log seq** automatically creates a new page with today's date.
6. **Adjust (optional)** — Change themes, font size, or bullet symbols in **Log seq**. Most people use **Log seq** defaults.
7. **Work normally** — **Log seq** activates automatically when you think in outlines or need an open-source Obsidian alternative.

## Understanding Log seq Core Components

**Log seq** is not just another markdown editor. It combines several essential tools for outline thinking:

- **Block reference engine** — Every bullet point in **Log seq** has a unique ID. You can reference any block from anywhere in **Log seq**.
- **Daily journal core** — **Log seq** treats time as a first-class dimension. Open today's page in **Log seq** with one click.
- **Hierarchical database** — Unlike Obsidian, **Log seq** stores relationships between bullets. Sub-points know their parents in **Log seq**.
- **Markdown compatibility** — Export from **Log seq** to standard `.md` files. Import existing notes into **Log seq** without conversion.
- **Outline folding** — Collapse entire sections in **Log seq** to focus on specific branches. Your **Log seq** workspace stays clean.
- **Global search** — Find any bullet point across all journals and pages in **Log seq**. Search results show the full context in **Log seq**.
- **Graph view** — See how your outlines connect in **Log seq**. Every reference becomes a visual link within **Log seq**.

All these work together seamlessly in **Log seq**. You never need to switch between separate outlining tools when using **Log seq**.

## Advanced Use Cases for Log seq

**Scenario 1: Morning journal with nested structure**
Open **Log seq**. Type today's date. Under it, create three bullets: "Mind", "Body", "Spirit". Indent sub-bullets under each. Write 50 words. **Log seq** saves everything automatically.

**Scenario 2: Writing a book outline**
Create a page "Novel Outline" in **Log seq**. List chapters as top-level bullets. Indent scenes under chapters. Indent beats under scenes. **Log seq** lets you fold completed chapters. Your **Log seq** outline stays readable even at 10000 bullets.

**Scenario 3: Managing a software project**
In **Log seq**, create "Sprint 24". Indent "Backend", "Frontend", "DevOps". Under "Backend", indent "Auth API", "Payment API". Under "Auth API", indent "- Implement JWT", "- Write tests". **Log seq** becomes your lightweight JIRA alternative.

**Scenario 4: On-call engineer with a USB stick**
Copy the portable **Log seq** folder to a USB drive. Plug into any work computer (no admin rights needed). Open your **Log seq** journal and review the last 30 days of incident notes. **Log seq** works instantly.

**Scenario 5: Research paper compilation**
Create "Climate Change" page in **Log seq**. Indent "Causes", "Effects", "Solutions". Under "Causes", indent "CO2", "Methane", "Deforestation". For "CO2", indent 20 bullet points with citations. **Log seq** handles depth better than Obsidian for this use case.

**Scenario 6: Meeting notes**
During a meeting, open **Log seq**. Type `- Discussion`. Indent `- John mentioned API delays`. Indent further `- Root cause: authentication`. After meeting, collapse all sub-points. Share the outline from **Log seq** as plain text. Everyone understands the structure because **Log seq** preserves hierarchy.
